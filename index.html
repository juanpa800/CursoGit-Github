Haciendo pruebas para conocer git y github

Este trabajo se está haciendo con fines educativos propios, los cuales me permitirán aprender git y github

Comandos vistos y aprendidos:

Comando

Función

  1. git init

    2. Inicia el proyecto, indica que la carpeta sera el origen de un proyecto.

  2. git status

    3. Muestra los cambios SOLO los realizados dentro del proyecto lo que permite ver si se hicieron modificaciones desde el ultimo commit, si un archivo ya se hizo git add entonces no se muestra, excepto si se usa el siguiente comando.

  3. git status -s

    4. Con git status -s Podemos visualizar TODOS los archivos del root y las carpetas, si la carpeta o archivo tiene un verde es que todo su contenido está salvado, y rojo es que aún no se salva.

  4. git config global user.username "name"

    5. Configuración del nombre del propietario del proyecto.

  5. git config global user.email "email"

    6. Configuración del email del propietario del proyecto.

  6. git config -l

    7. Muestra como está configurada la información del proyecto.

  7. git add <file>

    8. Salva un archivo modificado enviándolo al Staging Area

  8. git add .

    9. Envía todos los archivos modificados al Staging Area

  9. git commit -m "mensaje del commit"

    10. Realiza un commit, enviando todo nuestro staging area a local y guardando esta versión del proyecto.
    Si es la primera vez que se realiza toca configurar el username y el email con git config global

  10. git log --oneline

    11. Permite ver todos los commits realizados.

  11. git reset --head <ID del commit>

    12. Permite restaurar el proyecto a un commit anterior.
    NOTA: Se perderán los commits hasta donde se decida regresar.

  12. git commit --amend

    13. Cambia la descripción del último commit realizado usando VIM en la consola git bash

    :i modo edición
    Borrar esa descripción antigua
    Esc
    Luego comenzar a escribir
    Esc nuevamente
    :wq comando para guardar y salir

  13. git remote add origin <URL>

    14. Enlaza el proyecto en github previamente creado con el repositorio del proyecto

    probablemente tengas que hacer git push -u origin master

  14. git push origin master

    15. Envía mis cambios de local a mi repositorio en remoto en github.

  15. git pull

    16. Envía los cambios de remoto a mi repositorio local.

  16. git tag <nombre del tag> -m "mensaje"

    17. Crea un tag EN LOCAL
    Tag: versión alguna del proyecto 100% utilizable dentro de lo que contiene.

  17. git push --tags

    18. Sube el tag al repositorio en remoto

  18. git clone <URL>

    19. Crea una copia del proyecto de la rama master del link

  19. git branch <nombre>

    20. Crea una rama con el nombre a partir del punto en el que está el proyecto

  20. git branch

    21. Muestra las ramas del proyecto y señala la rama en la que estamos ubicados

  21. git checkout <nombre rama>

    22. Sirve para moverse entre ramas.

  22. git merge <rama a fusionar>

    23. Parados en la rama a la que queremos llevar los cambio, utilizamos este código para hacer merge con la rama descrita arriba, en caso de presentar conflictos se informará

  23. git branch -d <nombre rama a borrar>

    24. Borra la rama indicada, se recomienda hacer esto una vez que se hace merge exitosamente
